With recent 1.7u40 and Kryo 2.2.2 things have changed a bit. JDK has become somewhat faster, Kryo improved in many areas, however has some weird outliers.

If fast serialization is run with 'FSTConfiguration.preferSpeedOverSize=true' little performance gain is traded against significant output size increase. IMO, the best overall size/performance ratio is given by plain FST default configuration.
